一、毕业设计具体内容、目标和可能遇到的问题

**具体内容：**

本项目旨在设计并实现一个功能完善的志愿活动管理系统(VMS)，采用前后端分离架构，支持志愿者、志愿组织和系统管理员三种用户角色，实现志愿活动的全生命周期管理。

系统主要功能模块包括：

1. **多角色注册与登录系统**：支持志愿者注册、志愿组织注册（需审核）、统一登录认证，采用JWT实现身份验证和权限控制。

2. **组织审核管理**：系统管理员对志愿组织入驻申请进行审核，审核通过后组织方可发布活动。

3. **志愿活动管理**：组织发布志愿活动（含多岗位设置），管理员审核活动，活动状态流转（待审核→待启动→运行中→已结项）。

4. **志愿者报名与签到**：志愿者报名活动岗位，组织审核报名，志愿者签到签退，组织发放志愿时长。

5. **评价与积分系统**：组织对志愿者进行多维度评价（培训、协作、执行力），系统自动计算综合评分和积分。

6. **用户管理**：管理员管理用户账号状态，用户个人中心管理个人信息。

7. **系统配置**：行政区划级联选择、服务类别字典管理。

**目标：**

1. 完成系统需求分析、数据库设计、前后端代码开发。
2. 实现分布式微服务架构，将系统拆分为用户服务(8081)、活动服务(8083)、系统服务(8087)三个独立服务。
3. 完成各功能模块的单元测试和集成测试。
4. 编写完整的系统文档和使用说明。
5. 后续实现活动推荐算法，根据志愿者历史参与记录推荐合适活动。

**可能遇到的问题：**

1. **分布式服务间数据同步问题**：三个独立服务之间如何保证数据一致性，目前采用共享数据库方式规避。
2. **推荐算法实现**：基于志愿者历史数据（参与活动类别、地区偏好、评价记录）构建推荐模型，需要设计合理的特征工程。
3. **并发处理**：活动报名时岗位人数限制的并发控制，需要考虑防止超额报名。
4. **系统测试数据不足**：需要生成足够的测试数据验证推荐算法效果。



二、采取的研究方法、技术路线、实验方案及可行性分析

**研究方法：**

采用软件工程的敏捷开发方法，按照垂直切片方式逐步实现各功能模块，每个切片包含完整的后端API、前端页面和数据库设计，确保每个功能可独立运行和测试。

**技术路线：**

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Vite + Element Plus + Pinia | 响应式框架、构建工具、UI组件库、状态管理 |
| 后端 | Spring Boot 3 + MyBatis Plus | 快速开发框架、ORM增强工具 |
| 认证 | JWT + Spring Security | 无状态认证、权限控制 |
| 数据库 | MySQL 8.0 | 关系型数据库 |
| 架构 | 分布式独立服务 | 三服务独立部署，前端Proxy分发请求 |

**架构设计：**

系统采用分布式架构，物理拆分为三个独立服务：
- **vms-service-user (端口8081)**：用户认证、组织管理、文件上传
- **vms-service-activity (端口8083)**：活动管理、报名管理、签到、评价
- **vms-service-system (端口8087)**：字典管理、行政区划

前端通过Vite Proxy将不同前缀的API请求分发到对应后端端口，实现服务解耦。

**实验方案：**

1. **功能测试**：对每个API接口编写测试用例，验证业务逻辑正确性。
2. **集成测试**：模拟完整业务流程（注册→审核→发布活动→报名→签到→评价）。
3. **性能测试**：使用JMeter对关键接口进行压力测试。
4. **推荐算法验证**：生成200+测试活动数据，验证推荐结果合理性。

**可行性分析：**

1. **技术可行性**：所选技术栈均为成熟稳定的主流框架，有丰富的社区资源和文档支持。
2. **功能可行性**：系统功能设计参照实际志愿服务管理流程，业务逻辑清晰可实现。
3. **时间可行性**：目前已完成七个垂直切片的核心功能开发，后续推荐算法可在规定时间内完成。
4. **资源可行性**：开发环境已配置完成，测试数据可自动生成。
